# Roadmap

## Phase 0: Product Reframing

- 明确产品从“多平台分析平台”收口为“反黑灰情报采集平台”
- 定义统一的线索、实体、关系、案件对象模型
- 定义平台专属风险分析器边界

## Phase 1: Collection Control Plane

- 建立新的 Web 控制台结构
- 任务中心支持：
  - 多平台动态采集表单
  - 运行状态与日志
  - 按风险场景保存任务模板
- 数据集中心支持：
  - 采集批次归档
  - 抽样预览
  - 标签与专题挂接

## Phase 2: Signal Plane

- 通用风险信号提取器
  - 风险词 / 暗语 / 导流语
  - 联系方式 / 外链 / 跳转痕迹
  - 批量化内容相似度
  - 异常活跃度与发布时间
- 平台专属信号提取器
  - `xhs`: 评论引流、软广招募、内容模板复用
  - `dy/ks`: 视频矩阵同步、字幕/口播导流
  - `wb`: 统一话术扩散、异常传播簇
  - `zhihu`: 批量回答与专业包装引流
  - `xianyu`: 商品暗语、卖家模板复用、异常价格带

## Phase 3: Entity & Case Plane

- 实体中心
  - 账号
  - 卖家
  - 商品
  - 内容
  - 联系方式
- 关系图谱
  - 同文案
  - 同联系方式
  - 同图片 / 商品模板
  - 同时间线行为
- 案件中心
  - 线索归档
  - 研判备注
  - 风险结论

## Phase 4: Evidence & Alert Plane

- 证据包导出
- 时间线回放
- 风险告警与规则命中
- 报告生成与协作流转
