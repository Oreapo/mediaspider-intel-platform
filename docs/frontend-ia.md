# Frontend Information Architecture

## 顶层导航

顶层导航不应再按“通用分析平台”设计，而应按反黑灰工作流设计：

1. Dashboard
2. Tasks
3. Datasets
4. Signals
5. Entities
6. Cases
7. Analysis
8. Evidence
9. Logs
10. Settings

## 页面说明

## Dashboard

首页只放高价值摘要：

- 高风险平台与高风险主题
- 今日新增信号
- 待研判案件
- 最新数据集
- 最近告警

## Tasks

任务中心负责：

- 多平台任务列表
- 场景化任务模板
- 平台动态表单
- 运行状态
- 启停操作

布局建议：

- 左：任务列表 / 场景筛选
- 右：任务编辑器 / 运行详情

## Datasets

数据集中心负责：

- 采集批次列表
- 平台 / 时间 / 场景筛选
- 抽样预览
- 数据结构摘要
- 挂接到信号提取或案件

## Signals

信号中心负责：

- 风险词命中
- 联系方式 / 导流痕迹
- 模板复用命中
- 异常活跃度命中

建议支持：

- 规则筛选
- 风险等级排序
- 一键加入案件

## Entities

实体中心负责：

- 风险账号
- 风险卖家
- 风险商品
- 风险联系方式
- 关系图谱视图

## Cases

案件中心负责：

- 线索汇总
- 关联实体
- 研判结论
- 案件时间线
- 协作备注

## Analysis

分析中心负责：

- 通用风险分析
- 平台专属风险分析
- 跨平台关联分析

建议拆成二级标签：

- `Signals`
- `Platform`
- `Cross Platform`
- `Timeline`

## Evidence

证据中心负责：

- 证据快照列表
- 原始记录引用
- 证据包导出
- 报告导出

## Logs

日志中心负责：

- 当前运行日志
- 历史运行日志
- 错误诊断

## Settings

设置中心负责：

- 平台配置
- 登录态配置
- 存储配置
- 规则配置
- 告警配置

## 关键交互模式

### 模式一：线索驱动

创建任务 -> 执行采集 -> 生成数据集 -> 抽取风险信号 -> 加入案件

### 模式二：实体驱动

围绕某个账号 / 卖家 / 商品扩展采集 -> 生成关系图 -> 归档证据

### 模式三：案件驱动

围绕专题案件持续刷新任务、补齐数据集、更新结论和证据包
