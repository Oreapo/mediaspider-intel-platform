# 产品定位（收敛版）

生成日期：2026-06-30
状态：草案，待团队对齐后据此重排路线图

> 本文的目的只有一个：把"什么都能做一点"的平台，收敛成"为一类用户、一个场景做到最好"的产品。
> 在确认本文之前，不要再新增横向功能。

---

## 1. 目标用户（ICP）

**互联网企业的业务安全 / 反黑灰产 / 风控团队**（电商、社交、金融科技、游戏、内容平台等，凡是有"业务安全/反黑产"职能的公司）。

典型角色：

- **风控 / 反黑灰产分析师**（主用户，80% 时间在这工作）：每天盯社交/内容平台上针对本平台或本行业的黑灰产生态，研判线索、追踪团伙。
- **业务安全负责人**（决策者）：看态势、定监测策略、签处置。
- **处置 / 情报对接人**：拿研判结论去做拦截、封号、模型补样本、投诉、报案。

**不是**给品牌打假、舆情公关、MCN、执法机关——这些场景先全部排除。

---

## 2. 单一核心场景（先只赢这一个）

> **"社交/内容平台黑灰产生态的发现、团伙聚合与处置"**

反黑灰产分析师配置一批监测关键词（覆盖本平台/本行业的黑灰产活动），平台**持续采集**公开内容 →
**自动识别黑灰产招募/工具/服务信号** → 把分散的账号、联系方式、QQ/微信群、引流落地页**聚合成"团伙实体"** →
分析师研判确认 → 输出**处置研判材料**（拦截/封号名单、模型样本、投诉/报案依据）。

常见黑灰产监测对象（关键词族，按业务可裁剪）：

- **拉新作弊 / 薅羊毛**：刷单炒信、薅羊毛招募、众包砍价/助力、虚假拉新。
- **账号黑产**：批量注册养号、卡商/接码、账号交易、实名号买卖。
- **引流落地**：赌博/色情/诈骗/跨境导流、私域引流话术。
- **工具与服务**：外挂/脚本/群控、代过验证码、打码平台、代练代办。
- **数据黑产**：爬虫黑产、个人信息/数据买卖、API 滥用。

与品牌打假的关键区别（决定产品形态）：

| 维度 | 反黑灰产风控（本产品方向） | 品牌打假（已排除） |
|---|---|---|
| 核心价值 | **团伙关联**：多账号/多平台引流到同一联系方式/群/落地页 | 单点假货账号 |
| 输出物 | 拦截/封号名单、风控模型样本、处置/报案依据 | 维权投诉 |
| 时效性 | 强，黑产话术/工具迭代极快，需持续监测 | 较弱 |
| 合法性基础 | 企业自身业务安全/反黑产，相对清晰 | 较弱 |

## 3. 价值主张（一句话）

**"把社交平台上分散的黑灰产线索，自动聚合成可处置的团伙画像。"**

护城河不在"能爬"，而在两件事：

1. **信号识别质量**（黑灰产招募话术、工具/服务兜售、引流模式的识别准确率）——这是当前最薄弱、最该投入的地方。
2. **团伙聚合**（跨账号、跨平台、按联系方式/群组收敛成同一伙人）——这是风控比通用爬虫值钱的核心。

---

## 4. MVP 砍 / 留清单

聚焦上面那一个场景，重新切 MVP：

### ✅ 留（并做深）
- 监测任务：从"一次性采集"转向**持续监测规则**（关键词 + cron + 命中即出信号）。
- 黑灰产信号提取器：`risk_terms`、`contact_points`，**新增** `recruit_pattern`（招募话术）、`service_offer`（工具/服务兜售）、`traffic_route`（引流落地）。
- **团伙实体聚合**：账号 + 联系方式 + 群组 + 落地页 → 自动按联系方式/群组收敛，关系图。
- 处置/案件：输出处置研判材料。

### 🔻 降权（先别当主功能）
- i18n：**保留 zh-CN（简体）+ en-US（英文）两套，删除 zh-TW（繁体）**。主要用户是中文使用者、简体即可；英文留作对外/演示（用户 2026-07-01 决定）。新增前端文案需同步维护这两套并跑 check:i18n。
- 报告 Markdown 编辑器精修：够用即可。
- "情报分析"页的跨平台炫技分析。

### ⏸️ 改名 / 重新定性
- **"证据包" → "处置研判材料"**：去掉法律"出证"暗示，避免误导用户预期，也规避合规风险。

### ❌ 暂不做
- 多租户 / 对外 SaaS（确认是自用/单组织后再说）。
- 反爬最重的平台（小红书/抖音）优先级后置；**先做公开文本平台**（微博、贴吧、知乎）跑通闭环更现实。

## 5. 信息架构重构（核心交付）

当前导航是**"按对象组织"**——12+ 个顶级页面，每种业务对象一个页。分析师认知负担极高。
收敛为**"按任务阶段组织"**——5 个主导航，对应分析师真实工作流：

```
现在（对象中心，12+ 项）              收敛后（任务中心，5 项 + 系统区）
─────────────────────────            ─────────────────────────────────
首页看板                              ① 风险态势   （Dashboard 重做）
采集任务 / 任务详情                    ② 监测任务   （持续监测规则）
数据集 / 数据集详情      ┐            ③ 线索研判   （Signals 为核心工作台，
情报分析                ┘                          数据集/分析作为"来源追溯"内嵌）
风险信号 / 信号详情                    ④ 团伙画像   （Entities，关系图为重点）
风险实体                              ⑤ 处置与案件 （Cases，证据包/报告内嵌为"交付物"）
案件                    ┐
证据包                  ┘─ 合并        ── 系统区（非主导航）──
报告                                  设置 · 日志 · （情报分析降为信号页内工具）
日志 / 设置
```

**改动要点：**

1. **数据集、情报分析** 从顶级导航**移除**，降为"线索研判"页里的来源追溯/分析工具——它们是中间产物，不是分析师的目标。
2. **证据包、报告** 合并进"处置与案件"作为交付物 tab——分析师的目标是"处置"，不是"管理证据包"。
3. **首页(风险态势)** 从通用看板重做为风控态势：今日新增欺诈信号、待研判高危、活跃团伙 Top、平台分布、监测覆盖率。
4. 主导航从 12+ → **5 项**，认知负担大幅下降。

## 6. 成功指标（PM 需要先定义"赢"长什么样）

现在产品**无法回答"信号有没有用"**。上线前必须埋这几个指标：

- **信号准确率（precision）**：研判确认 / 总信号 —— 直接衡量核心价值。
- **团伙发现数 / 处置数**：产出。
- **信号 → 处置时长**：效率。
- **监测覆盖**：关键词命中率、平台覆盖。

---

## 7. 合规底线（即使是自用也要做）

风控/反欺诈的合法性基础相对清晰，但仍受 PIPL 约束，至少要做到：

- **目的限制**：只用于本企业反欺诈，不做无关画像。
- **最小化**：只采公开信息中与欺诈识别相关的字段。
- **留存期限**：处置后定期清理原始数据。
- **去"证据"化**：交付物定性为内部研判材料。
