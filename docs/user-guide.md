# MediaSpider 情报平台使用说明

本文档面向平台使用者，说明各功能模块的用途、典型操作流程和注意事项。平台的核心工作流是：

采集任务 -> 数据集 -> 风险信号 -> 风险实体 -> 案件 -> 证据包 / 报告。

## 1. 本地启动

首次运行可在项目根目录创建本地配置：

```powershell
Copy-Item .env.example .env
```

后端启动器与 Vite 会共同读取这个 `.env`。终端中已设置的同名环境变量优先级更高。

### 后端服务

在项目根目录运行：

```powershell
.venv\Scripts\python.exe -m backend.app
```

如果虚拟环境路径不同，可使用：

```powershell
python -m backend.app
```

默认后端地址：

```text
http://127.0.0.1:8180
```

接口文档地址：

```text
http://127.0.0.1:8180/docs
```

本地启动入口读取 `MEDIASPIDER_BACKEND_HOST`、`MEDIASPIDER_BACKEND_PORT` 和
`MEDIASPIDER_BACKEND_RELOAD`。Docker 部署仍使用容器端口 `8000`。

### 前端服务

在另一个终端运行：

```powershell
cd frontend
npm install
npm run dev
```

默认前端地址：

```text
http://127.0.0.1:5200
```

## 2. 首页看板

入口：`首页看板`

首页看板用于快速判断平台当前状态。打开平台后建议先查看这里。

主要内容：

- 采集健康：展示任务数量和失败运行情况。
- 复核压力：展示待处理高风险信号数量。
- 案件推进：展示进行中案件和待出证案件。
- 快捷入口：快速进入采集任务、风险信号、案件和报告页面。
- 工作看板：按采集队列、信号复核、案件调查、交付输出展示当前待处理事项。
- 风险分布：按平台和场景统计高风险信号、数据集、实体和案件。

典型用法：

1. 查看是否有失败采集运行。
2. 查看是否有高风险信号等待复核。
3. 查看是否有案件已经可以生成证据包。
4. 从快捷入口进入对应页面继续处理。

## 3. 采集任务

入口：`采集任务`

采集任务是平台数据流的起点，用于配置不同平台的采集目标、运行参数和信号提取策略。

主要功能：

- 创建采集任务。
- 按平台、状态、模式、实体类型、风险场景筛选任务。
- 进入任务详情页查看配置、运行记录、关联数据集和采集器诊断。
- 启用或停用任务。
- 手动启动采集运行。
- 诊断采集器环境。
- 扫描和执行定时任务。
- 查看采集执行槽、等待优先级和跨进程运行租约状态。

创建任务时需要填写：

- 任务名称：便于后续识别。
- 平台：例如小红书、抖音、微博、闲鱼等。
- 实体类型：内容、账号、商品等。
- 风险场景：引流导流、灰产招募、欺诈推广、卖家风险、商品风险、主题监测等。
- 任务模式：关键词搜索、详情采集、创作者采集等。
- 输入内容：关键词、详情 ID、URL 或创作者主页。
- 信号提取器：例如 `risk_terms,contact_points,template_similarity`。

建议流程：

1. 创建任务并填写关键词或目标链接。
2. 点击“诊断采集器”，确认本地采集依赖可用。
3. 点击“启动采集运行”。
4. 进入任务详情页查看运行记录和自动生成的数据集。
5. 采集完成后前往“数据集”查看结果。

“启动采集运行”会立即返回已提交的运行记录，不会等待采集器结束。任务详情页会在运行处于
`pending` 或 `running` 时自动刷新，并展示排队、采集和结果入库进度；完成、失败或取消后停止轮询并同步关联数据集。
失败运行还会显示错误分类、是否建议直接重试，以及检查登录态、采集配置、网络、代理、存储或运行日志等处理建议。
对于标记为“建议重试”的失败，可在运行记录中直接重新提交；新运行会保留来源运行 ID 和重试次数，方便追踪完整链路。

## 4. 数据集

入口：`数据集`

数据集用于沉淀采集结果，是后续分析和风险信号提取的标准输入。

主要功能：

- 创建数据集记录。
- 按关键词、平台、类型、场景、标签筛选数据集。
- 预览数据集内容。
- 进入数据集详情页查看来源任务、来源运行、预览数据和分析输出。
- 删除数据集记录。

数据集字段说明：

- 数据集名称：用于识别数据来源和批次。
- 来源平台：数据来自哪个平台。
- 数据集类型：原始数据、标准化数据、可分析数据。
- 风险场景：该批数据服务于哪个调查场景。
- 标签：用于归类项目、品牌、活动或专题。
- 存储路径：相对 `backend/storage/dataset_files` 的数据文件路径。

典型用法：

1. 从采集任务生成数据集，或手动登记已有数据文件。
2. 点击“预览”或进入详情页确认数据字段和样本内容。
3. 在详情页追溯来源任务和来源运行。
4. 前往“风险信号”从数据集中提取线索。

## 5. 情报分析

入口：`情报分析`

情报分析用于对数据集进行通用分析、平台分析或跨平台分析。

主要功能：

- 创建分析任务。
- 查看分析任务状态。
- 查看分析输出。

分析范围：

- 通用分析：适合摘要、关键词、主题统计。
- 平台分析：针对单一平台特征进行分析。
- 跨平台分析：用于关联多个平台的数据和风险对象。

典型用法：

1. 选择一个数据集。
2. 选择分析范围。
3. 输入分析类型，例如 `summary`、`topic_map`、`seller_profile`。
4. 填写参数 JSON。
5. 创建分析任务并查看输出。

## 6. 风险信号

入口：`风险信号`

风险信号是平台的核心复核对象，代表从数据集中提取出的可疑线索。

主要功能：

- 从数据集自动提取风险信号。
- 创建人工风险信号。
- 按数据集、状态、风险等级、类型筛选信号。
- 查看信号来源追溯。
- 进入信号详情页追溯来源数据集、采集任务、运行记录和关联案件。
- 将信号标记为复核中、已确认或已忽略。
- 删除信号。

风险等级：

- 低
- 中
- 高
- 严重

信号状态：

- 新建：刚生成，尚未处理。
- 复核中：正在人工判断。
- 已确认：确认有价值，可进入实体或案件流程。
- 已忽略：暂不处理或判断无效。

建议流程：

1. 选择数据集并配置提取器。
2. 点击“提取信号”。
3. 优先处理高风险和严重信号。
4. 进入信号详情页核对证据字段、来源行和关联案件。
5. 对有效信号点击“确认”。
6. 前往“风险实体”从确认信号中沉淀实体。

## 7. 风险实体

入口：`风险实体`

风险实体用于把多个信号聚合成可持续追踪的对象，例如账号、卖家、商品、联系方式等。

主要功能：

- 从已确认信号创建实体。
- 手动创建实体。
- 创建实体之间的关系。
- 合并重复实体。
- 按关键词、平台、类型、状态、最低风险分筛选实体。
- 查看实体详情、关联系号、关联数据集和关系。
- 在实体详情中查看关系图，中心节点是当前实体，粗线表示高置信关系。

典型实体类型：

- 账号
- 内容
- 商品
- 卖家
- 联系方式
- 群组
- 链接

建议流程：

1. 在“风险信号”确认有效信号。
2. 在“风险实体”中从确认信号创建实体。
3. 对同一对象的多个实体进行合并。
4. 创建实体关系，例如账号关联联系方式、卖家关联商品。
5. 将关键实体挂接到案件。

## 8. 案件

入口：`案件`

案件是调查容器，用于组织数据集、信号、实体、分析输出、备注和证据包。

主要功能：

- 创建案件。
- 关联数据集、信号、实体或分析输出。
- 添加调查备注。
- 按状态、优先级、案件类型、负责人筛选案件。
- 查看案件详情和时间线。
- 在案件详情中查看证据树，按数据集、信号、实体、分析输出和证据包组织证据结构。
- 将案件标记为调查中、待出证或已关闭。
- 从案件生成证据包。

案件状态：

- 打开：案件已创建，等待处理。
- 调查中：正在持续补充线索和研判。
- 待出证：材料已经基本齐备，可以生成证据包。
- 已关闭：案件处理完成。

建议流程：

1. 创建案件并填写背景摘要。
2. 将相关数据集、信号、实体或分析输出关联到案件。
3. 添加调查备注，记录判断依据和下一步动作。
4. 案件详情会按时间展示初始状态及后续状态流转，并保留操作者信息。
5. 在证据树中核对案件材料是否完整。
6. 案件材料齐备后标记为“待出证”。
7. 生成证据包或报告。

## 9. 证据包

入口：`证据包`

证据包用于把案件相关材料打包成可追溯、可下载的交付物。

主要功能：

- 从案件生成证据包。
- 查看证据包清单。
- 下载证据包。
- 删除证据包。

证据包通常包含：

- 案件基本信息。
- 关联数据集。
- 关联风险信号。
- 关联风险实体。
- 调查备注。
- 来源记录和 manifest 清单。

建议流程：

1. 在案件中确认材料齐备。
2. 在“证据包”页面选择案件。
3. 输入证据包名称或使用默认名称。
4. 生成后查看清单详情。
5. 下载证据包用于归档或交付。

## 10. 报告

入口：`报告`

报告用于把案件和分析结果整理成可阅读的情报文档。

主要功能：

- 从案件生成报告。
- 查看报告列表。
- 使用报告编辑器修改标题、状态和 Markdown 内容。
- 预览报告内容。
- 下载报告。
- 删除报告。

报告类型可根据业务场景扩展，例如：

- 案件摘要报告
- 风险研判报告
- 证据交付报告
- 平台专题报告

建议流程：

1. 选择一个案件。
2. 填写报告类型和报告名称。
3. 点击“生成报告”。
4. 使用报告编辑器补充人工结论、修订证据描述或调整状态。
5. 在报告预览区检查内容。
6. 下载报告用于交付。

## 11. 日志

入口：`日志`

日志页面用于排查采集运行和任务执行情况。

主要功能：

- 查看运行总数、成功数、失败数、日志数量。
- 查看任务运行历史。
- 查看单次运行日志输出。
- 刷新日志列表。

典型用法：

1. 当采集任务失败时，先进入“日志”页面。
2. 找到失败运行记录。
3. 查看日志输出和错误信息。
4. 返回“采集任务”页面调整配置或重新运行。

## 12. 设置

入口：`设置`

设置页面用于配置平台登入 Profile、通知规则、站内通知中心和查看通知投递记录。

主要功能：

- 创建平台登入 Profile。
- 诊断 Profile 是否具备运行条件。
- 更新或删除 Profile 凭据。
- 创建通知规则。
- 启用或停用规则。
- 删除规则。
- 手动执行定时摘要。
- 查看站内通知中心。
- 将站内通知标记为已读或未读。
- 一键标记全部站内通知为已读。
- 查看通知投递记录。
- 按状态、渠道、目标类型筛选投递记录。
- 对失败投递执行重试，并查看重试次数和最近尝试时间。

平台登入 Profile 字段说明：

- 平台：Profile 适用的平台，例如小红书、抖音、闲鱼。
- 认证类型：`cookie` 适合自动化采集；`qrcode` 和 `phone` 需要交互登录。旧版 `state_file` Profile 会保留，但 MediaCrawler CLI 不支持传入任意状态文件，因此不能用于新任务。
- credentials_ref：Cookie 文本，页面会脱敏显示。

### MediaCrawler Web 运行环境

当前 Web 后端只需要 MediaCrawler 核心运行时，不需要安装原桌面项目的 PySide6 GUI 依赖。为 MediaCrawler 源码目录创建独立虚拟环境后，使用本项目的精简依赖清单安装：

```powershell
python -m venv <MediaCrawler>\.venv
<MediaCrawler>\.venv\Scripts\python.exe -m pip install -r backend\requirements-mediacrawler.txt
<MediaCrawler>\.venv\Scripts\python.exe -m playwright install chromium
```

任务诊断会优先使用 `<MediaCrawler>\.venv`，不存在时才尝试系统 `uv`。
- 运行设置 JSON：会在任务运行时注入 `runtime_payload_json`，常用字段包括 `headless`、`max_concurrency_num`、`timeout_seconds`、`queue_priority`。队列优先级支持 `critical`、`high`、`normal`、`low`，同优先级按进入队列的先后顺序执行。

Profile 使用流程：

1. 在“设置”页面创建平台登入 Profile。
2. 点击“诊断”确认 Profile ready 为 true。
3. 在创建或更新采集任务时填写该 Profile 的 `auth_profile_id`。
4. 任务运行时会自动把 Profile 中的登录态和运行设置注入采集器。
5. Cookie 失效后，在 Profile 列表点击“更新凭据”进行轮换。

通知规则字段说明：

- 规则名称：便于识别规则用途。
- 风险阈值：只通知达到指定风险等级的对象。
- Cron 表达式：控制定时执行频率。
- 场景：限制通知适用的风险场景。
- 平台：限制通知适用的平台。
- 渠道：例如站内、邮件、Webhook。
- 冷却分钟数：避免短时间重复通知。
- 邮件接收人：邮件通知目标。
- Webhook 地址：外部系统接收地址。

邮件渠道配置：

邮件渠道使用 `channel_config_json` 中的 SMTP 字段发送摘要邮件。常用字段如下：

- `email_recipients`：收件人列表，也可用逗号分隔字符串。
- `smtp_host`：SMTP 服务器地址。
- `smtp_port`：SMTP 端口，默认 `25`。
- `smtp_from`：发件人地址。
- `smtp_username`：SMTP 用户名，可选。
- `smtp_password`：SMTP 密码，可选。
- `smtp_use_tls`：是否启用 STARTTLS，默认 `false`。
- `smtp_timeout_seconds`：SMTP 连接超时秒数，默认 `10`。

示例：

```json
{
  "email_recipients": ["risk@example.com"],
  "smtp_host": "smtp.example.com",
  "smtp_port": 587,
  "smtp_from": "platform@example.com",
  "smtp_username": "platform@example.com",
  "smtp_password": "your-password",
  "smtp_use_tls": true
}
```

Webhook 渠道配置：

Webhook 渠道使用 `webhook_url` 向外部系统发送 JSON 摘要。

```json
{
  "webhook_url": "https://example.com/webhook"
}
```

站内通知中心说明：

- `internal_inbox` 渠道命中的通知会进入站内通知中心。
- 未读通知会显示为未读状态，可单条标记为已读或未读。
- “全部已读”用于快速清理当前站内收件箱。
- 可按关键词搜索通知标题、规则、目标对象或摘要。

建议流程：

1. 创建高风险通知规则。
2. 设置风险阈值和平台/场景过滤条件。
3. 配置渠道。
4. 点击“立即执行定时任务”验证效果。
5. 在站内通知中心处理 internal_inbox 通知。
6. 在投递记录中查看发送、失败或跳过情况。
7. 如果投递失败，修正 SMTP 或 Webhook 配置后，在投递记录中点击“重试投递”。

## 13. 推荐业务流程

### 日常监控流程

1. 打开首页看板。
2. 查看采集健康和复核压力。
3. 优先处理失败采集运行。
4. 复核高风险信号。
5. 推进待出证案件。

### 新专题调查流程

1. 在“采集任务”创建专题采集任务。
2. 运行任务并生成数据集。
3. 在“数据集”预览结果。
4. 在“风险信号”提取信号。
5. 确认有效信号。
6. 在“风险实体”沉淀对象和关系。
7. 在“案件”创建专题案件并关联材料。
8. 生成证据包和报告。

### 交付流程

1. 确认案件状态为“待出证”。
2. 生成证据包。
3. 生成报告。
4. 下载材料。
5. 关闭案件。

## 14. 常见问题

### 看板没有数据

可能原因：

- 尚未创建采集任务。
- 尚未生成数据集。
- 尚未提取风险信号。

建议先从“采集任务”创建并运行任务。

### 采集运行失败

建议检查：

- 是否已经配置 MediaCrawler 根目录。
- 登录态或 Cookie 是否有效。
- 采集目标是否可访问。
- 任务输入是否为空。
- 日志页面是否有更具体的错误。

### 数据集无法预览

建议检查：

- 存储路径是否正确。
- 文件是否存在于后端数据目录。
- 文件格式是否符合平台预览逻辑。

### 信号提取结果为空

可能原因：

- 数据集中没有命中规则的内容。
- 提取器名称填写错误。
- 数据集字段结构与提取器预期不一致。

可以先创建人工信号验证后续流程。

### 无法生成证据包或报告

建议检查：

- 是否已经创建案件。
- 案件是否关联了数据集、信号或实体。
- 后端存储目录是否可写。
